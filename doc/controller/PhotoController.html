PhotoController Class: In-depth Guide

The PhotoController class facilitates detailed interactions with photos in an album, such as viewing enlarged photos, editing tags, and navigating through photos.

Detailed Method Descriptions

Initialization and Display

void start(ArrayList<User> users, ListView<Photo> photos, User user, Album selectedAlbum)

Prepares the photo view by loading selected album photos and user information. It sets up the environment for photo manipulation and tagging functionalities.

Photo Navigation

void handleSwitchButton(ActionEvent event)

Based on user interaction, navigates to either the next or previous photo in the album, enabling a seamless photo browsing experience within the album.

Tag Management

void handleAddTagButton()

Activates input fields for the addition of new photo tags, enhancing photo metadata and searchability within the application.

void handleDeleteTagButton(ActionEvent event)

Removes an existing tag from the photo, updating the application's data layer to reflect these changes immediately.

void handleConfirmButton()

Confirms the addition of a new tag, ensuring no duplicates, and updates the photo's metadata accordingly.

void handleCancelTagButton(ActionEvent event)

Aborts the current tagging operation, reverting any unsaved changes made by the user.

Application Navigation

void handleBackButton(ActionEvent event)

Returns the user to the broader album view, maintaining the application's navigational flow and user context.

void handleLogoutButton(ActionEvent event)

Secures the application by logging out the current user, ensuring data privacy and security upon user exit.

Practical Usage Examples

Within the photo view, users can leverage PhotoController functionalities to enrich their photo viewing experience. For instance, adding geographical and thematic tags to photos allows for better organization and later retrieval.
