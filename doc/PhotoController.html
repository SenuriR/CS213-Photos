PhotoController Class

The PhotoController class enables users to interact with individual photos within an album, providing functionalities such as viewing, tagging, and navigation between photos.

Key Functionalities and Methods

  • void start(ArrayList<User> users, ListView<Photo> photos, User user, Album selectedAlbum) - Initializes the controller with the list of users, selected album's photos, and the currently logged-in user.
  • void handleSwitchButton(ActionEvent event) - Navigates to the next or previous photo in the album based on the button pressed.
  • void handleAddTagButton() - Enables text fields for entering a new tag's type and value.
  • void handleDeleteTagButton(ActionEvent event) - Removes the selected tag from the photo and updates the data storage.
  • void handleConfirmButton() - Adds a new tag to the photo if it doesn't already exist, then updates the data storage.
  • void handleCancelTagButton(ActionEvent event) - Cancels the tag addition or deletion operation.
  • void handleBackButton(ActionEvent event) - Returns the user to the album view.
  • void handleLogoutButton(ActionEvent event) - Logs out the current user and returns to the login screen.

Usage

The PhotoController is primarily used within the photo view scene. It is responsible for all interactions related to a single photo, including displaying the photo, managing tags, and navigating among the photos in an album.

Example:

When a user selects a photo within an album, the PhotoController is invoked to display the photo along with its details and tags. The user can then add new tags, delete existing ones, or navigate to other photos within the same album.
